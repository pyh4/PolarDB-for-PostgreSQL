PolarDB for PostgreSQL

A cloud-native database service independently developed by Alibaba Cloud

Getting Started Architecture Introduction

Flexible Scalability

Scale out compute/storage clusters on demand.

Millisecond-level Latency

Lazy/parallel replay via shared-storage-based WAL and LogIndex.

HTAP

Shared-storage-based massively parallel processing (MPP) framework.

# Use with Docker

Pull the instance imageopen in new window of PolarDB for PostgreSQL based on local storage. Create, run and enter the container, and use PolarDB instance directly:

# pull the instance image from DockerHub
docker pull polardb/polardb_pg_local_instance:single
# create, run and enter the container
docker run -it --cap-add=SYS_PTRACE --privileged=true --name polardb_pg_single polardb/polardb_pg_local_instance:single bash
# check
psql -h 127.0.0.1 -c 'select version();'

# Develop with Docker

Pull the development imageopen in new window of PolarDB for PostgreSQL from DockerHub. Create, run and enter the container:

# pull the development image of PolarDB
docker pull polardb/polardb_pg_devel
# create, run and enter the container
docker run -it --cap-add=SYS_PTRACE --privileged=true --name polardb_pg_devel polardb/polardb_pg_devel bash

After entering the container, clone the latest stable code from GitHub, build and deploy the simplest PolarDB instance and check:

# code fetching
git clone -b POLARDB_11_STABLE https://github.com/ApsaraDB/PolarDB-for-PostgreSQL.git
cd PolarDB-for-PostgreSQL
# build and deploy
./polardb_build.sh
# check
psql -h 127.0.0.1 -c 'select version();'
            version
--------------------------------
 PostgreSQL 11.9 (POLARDB 11.9)
(1 row)
